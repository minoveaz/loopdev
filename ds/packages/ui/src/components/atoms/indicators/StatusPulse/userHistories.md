# Historias de Usuario: StatusPulse

Este átomo es la unidad mínima de actividad y telemetría visual en LoopDev OS. Indica que un sistema, proceso o contexto está "vivo" y operativo.

## Bloque 0: ADN de Composición
1. **Trinidad Cromática:** 
   - Verde (`success`): Sistema operativo / saludable.
   - Amarillo (`energy`): Contexto activo (Momentum) / Advertencia.
   - Rojo (`danger`): Error crítico / Desconexión.
   - Azul (`primary`): Proceso de sistema / Información.
2. **Sintaxis Loop:** Debe ser capaz de vivir dentro de contenedores con brackets `{ }`.
3. **Technical Canvas:** El punto debe tener un resplandor técnico (glow) que use el color de la variante.
4. **Surface Soul:** Animación de pulso rítmica constante (Momentum).

## Historias de Usuario
- **Historia 1 (Indicador de Salud):** Como usuario, quiero ver un punto verde parpadeante junto al estado del sistema para confirmar que la plataforma está conectada.
- **Historia 2 (Momentum de Contexto):** Como usuario, quiero que el módulo activo en el sidebar muestre un pulso amarillo para identificar mi foco actual.
- **Historia 3 (Accesibilidad de Movimiento):** Como sistema, quiero desactivar la animación automáticamente si el usuario ha configurado "reducir movimiento" en su OS.

## Casos de Estrés
- **Densidad Extrema:** El pulso debe ser visible y nítido incluso en tamaño micro (4px).
- **Contraste de Fondo:** El resplandor (glow) debe ser perceptible tanto en modo claro como oscuro.
